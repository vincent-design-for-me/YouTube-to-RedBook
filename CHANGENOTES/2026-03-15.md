# 2026-03-15 — 小红书内容生成 Pipeline

## 概述
从 YouTube 字幕提取工具扩展为完整的内容生成 pipeline：YouTube URL → 提取字幕 → LLM 分析关键知识点 → 生成小红书文案 → 生成中文信息海报配图。

## 架构决策

### 两阶段生成流程
- **Phase 1**: 字幕 → LLM 提取知识点 → LLM 生成小红书文案 → 用户预览
- **Phase 2**: 用户确认文案后 → 为每个知识点生成中文信息海报
- 原因：文案需要人工审核，图片生成耗时长且消耗 API 额度，不应自动触发

### LLM 决定知识点和图片数量
- 知识点数量不硬编码，由 LLM 根据视频内容丰富度自行判断（3~8+）
- 图片数量 = 知识点数量，每个知识点生成 1 张海报

### LLM Provider 抽象
- 默认使用 Gemini（`@google/genai` SDK）
- 配置 `LLM_BASE_URL` 后自动切换为 OpenAI 兼容接口（fetch POST `/chat/completions`）
- 切换 provider 只需改 `.env.local`，无需改代码

### 图片生成上下文
- 每张图的 prompt 包含完整字幕文本作为上下文，让 LLM 理解知识点的完整语境
- 不是只传知识点本身，避免 LLM 对知识点理解不足

### 本地存储（Supabase-ready）
- 生成结果存 `data/generations/{id}.json` + 图片存 `data/images/{id}/`
- JSON schema 对齐 Supabase 表结构，迁移时可直接导入
